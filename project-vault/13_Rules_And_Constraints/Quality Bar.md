# Quality Bar

## Product Quality

Every page should pass this test within 10 seconds:

* Does it feel luxurious?
* Does it feel trustworthy?
* Does it show expertise?
* Does it feel warm?
* Does it create aspiration?

## UX Quality

* Users should always understand where to go next.
* CTAs should be visible, clear, and mobile-friendly.
* The site should encourage exploration before product inquiry.
* Empty, loading, and error states should be graceful.

## UI Quality

* Generous whitespace.
* Large, relevant imagery.
* Short, meaningful copy.
* Clear visual hierarchy.
* No crowded sections.
* No template-heavy layouts.

## Engineering Quality

* Type-safe.
* Small, reusable components.
* No giant components.
* No duplicate components.
* Consistent API responses.
* Server-side validation.
* Mobile performance first.

## Performance Quality

* Homepage load target under 2 seconds.
* Lighthouse target 90+.
* Use optimized images.
* Avoid heavy client-side state.
* Lazy-load heavy animation/3D where possible.
