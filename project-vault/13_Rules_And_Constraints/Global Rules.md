# Global Rules

## Product Rules

* The site is a premium digital showroom, not ecommerce.
* Inspiration comes before products.
* Products support spaces.
* Sumanglam must not feel like a generic hardware store.
* The main business goal is qualified showroom visits, consultation requests, and WhatsApp inquiries.

## Brand Rules

* Nolte is a premium German kitchen solution brand.
* Mrida is Sumanglam's in-house solution brand for modular kitchens, modular wardrobes, and customized interior solutions.
* Wardrobes belong to Mrida.
* Do not treat Nolte and Mrida as identical.
* Hettich is parent to Blaupunkt.

## Architecture Rules

* Follow the documented architecture.
* Use Next.js App Router, TypeScript, Tailwind CSS, shadcn/ui, Prisma, PostgreSQL, Cloudinary.
* Use Auth.js for admin authentication.
* Use Google Analytics 4 for analytics.
* Three.js, GSAP ScrollTrigger, and Lenis.js are user-approved stack extensions.

## Design Rules

* Follow [[Homepage UX Specification]], [[Visual Style]], and [[Interaction Patterns]].
* Priorities: Inspiration, Trust, Luxury, Expertise.
* Mobile-first always.
* Use large imagery, generous whitespace, short copy, and clear hierarchy.

## Development Rules

* Prefer Server Components.
* Use Server Actions where appropriate.
* Maintain type safety.
* Build reusable components.
* Avoid duplication.
* Keep changes scoped.

## Database/API Rules

* Follow [[Database Overview]].
* Follow [[API Overview]].
* Do not introduce schema changes without updating documentation.
* Do not create undocumented endpoints.
* Use consistent API response structures.

## Documentation Rules

Whenever architecture changes, update documentation first and code second.
