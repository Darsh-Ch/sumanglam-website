# Build Readiness Notes

## Ready To Build

The docs are sufficient to start:

* Project scaffold.
* Design system foundation.
* Homepage layout.
* Core Prisma schema draft.
* Public read API skeletons.
* Consultation form/API skeleton.
* Admin auth planning.

## Do Not Start Without Resolving

Resolve before final production design:

* Exact color tokens.
* Exact font choices.
* Official imagery/brand assets.
* WhatsApp number.
* Business contact details and hours.

Resolve before production backend:

* Supabase vs Neon.
* Auth.js provider configuration.
* Notification channel.
* Validation library.
* Media storage shape.

Resolve before final navigation:

* Wardrobes route/nav placement.

Resolve before PDP finalization:

* Exact pricing vs price range.
* V1 product count.

## First Build Recommendation

Start with foundation and design system, then complete the homepage vertically. Do not build every route skeleton first.
