# Content Governance

## Required Content Status

Every content item should support:

* Draft.
* Published.
* Archived.

## Required Metadata

Every content item should support:

* SEO Metadata.
* Open Graph Metadata.
* Last Updated Date.
* Author / Editor.

## Admin Implications

Admin tools should be able to create and update:

* Inspirations.
* Brands.
* Products.
* Collections.
* Showroom content.

Delete behavior is documented as archive for admin APIs.

## Rules

* Do not hardcode content that belongs in the model.
* Do not publish orphaned content.
* Do not create categories or collections casually.
* Update documentation before changing architecture.
